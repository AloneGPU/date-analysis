# 🎯 ESP8266 分贝检测工具 - 完整指南

## 📋 项目概述

为你创建了**多个方案**的分贝检测工具，根据你的硬件选择最适合的方案：

### ⭐ 推荐方案：ESP32 + INMP441（最佳性能）
- 文件：`esp32/esp32_inmp441/esp32_inmp441.ino`
- 完整的I2S支持
- 高精度数字处理
- 推荐使用

### 🔧 备选方案1：ESP8266 + MAX4466（模拟）
- 文件：`esp8266/esp8266_inmp441_simple/esp8266_inmp441_simple.ino`
- 更兼容ESP8266
- 使用模拟输入
- 性能适中

### 🧪 测试方案：ESP8266 + 模拟数据
- 文件：`esp8266/esp8266_inmp441_simple/esp8266_inmp441_simple.ino`
- 设置 `useSimulatedData = true`
- 无需硬件，快速测试

---

## 🔌 硬件连接

### 方案1：ESP32 + INMP441（推荐）

```
INMP441      ESP32
VDD       →  3.3V
GND       →  GND
L/R       →  GND (左声道)
WS        →  GPIO25
SCK       →  GPIO26
SD        →  GPIO32
```

### 方案2：ESP8266 + MAX4466

```
MAX4466      ESP8266
OUT       →  A0
VCC       →  3.3V
GND       →  GND
```

## 🚀 快速开始

### 第一步：选择方案

| 你有什么硬件 | 推荐方案 |
|------------|---------|
| ESP32 + INMP441 | 使用 ESP32 版本（最佳） |
| ESP8266 + MAX4466 | 使用 ESP8266 简化版（模拟） |
| 只有ESP8266 | 先使用模拟数据模式测试 |

### 第二步：配置WiFi

修改相应版本代码中的WiFi信息：
```cpp
const char* WIFI_SSID = "你的WiFi名称";
const char* WIFI_PASSWORD = "你的WiFi密码";
```

### 第三步：上传代码

使用Arduino IDE上传到你的开发板

### 第四步：连接到Flutter应用

1. 打开你的数据监控应用
2. 选择"WiFi"模式
3. 扫描设备或手动输入IP
4. 开始查看实时分贝数据！

## 📊 数据格式

### 发送到Flutter应用的JSON：
```json
{
  "device_id": "esp32_db_001",
  "timestamp": 1234567890,
  "data": {
    "decibel": 55.3,
    "avg_decibel": 54.2,
    "max_decibel": 72.5,
    "min_decibel": 35.1,
    "level": "正常"
  }
}
```

```
[分贝] 当前: 55.3 dB | 正常 | 平均: 54.2 dB | 最大: 72.5 dB | 最小: 35.1 dB
```

## 🔍 故障排除

### 问题1：找不到代码文件

**解决**：查看上面的文件结构，我已创建在相应目录

### 问题2：ESP8266的INMP441不工作

**原因**：ESP8266对I2S支持有限

**解决**：
1. 改用ESP32
2. 或使用MAX4466模拟方案
3. 先使用模拟数据模式测试

### 问题3：Flutter应用收不到数据

**解决**：
1. 确认在同一WiFi
2. 检查防火墙
3. 手动输入IP连接
4. 查看串口监视器输出
