## 📱 基本使用流程

### 步骤 1：连接设备

1. 点击右上角的 ⚙️ **设置** 图标
2. 选择通讯方式：
   - **MQTT**：适合远程数据传输
   - **蓝牙**：适合近距离通信
   - **WiFi**：适合局域网内通信，可扫描支持协议的数据采集设备
3. 填写配置信息；WiFi 方式也可以点击 **扫描局域网设备** 后选择设备自动填入
4. 点击 **连接** 按钮

### 步骤 2：查看数据

1. 连接成功后，返回主界面
2. 查看实时的数据卡片
3. 点击 **查看更多图表** 查看更多可视化
4. 在图表页面切换不同的图表类型

### 步骤 3：管理数据

1. 点击 💾 **存储设置** 图标
2. 查看数据统计信息
3. 开启/关闭自动保存
4. 导出数据或清理历史数据

## 📊 数据格式示例

你的单片机或嵌入式设备需要发送以下格式的数据：

```json
{
  "device_id": "sensor_001",
  "timestamp": 1640000000000,
  "data": {
    "temperature": 25.5,
    "humidity": 60.0,
    "voltage": 3.3,
    "current": 0.5
  }
}
```

**说明：**
- `device_id`：设备唯一标识
- `timestamp`：毫秒级时间戳
- `data`：传感器数据，支持多个字段

## 🔧 MQTT 配置示例

```
Broker: broker.emqx.io
端口: 1883
Topic: sensor/data
用户名: (可选)
密码: (可选)
```

## 📡 WiFi 配置示例

```
主机地址: 192.168.1.100
端口: 8080
```

WiFi 扫描使用 UDP 局域网发现协议，只会显示按协议应答的数据采集设备，不会扫描普通 WiFi 热点。固件端对接说明见 [WiFi 发现协议](docs/wifi_discovery_protocol.md)。

## ⚡ 常见问题

### Q: 连接失败怎么办？
**A:** 检查以下几点：
- 网络连接是否正常
- 设备地址和端口是否正确
- MQTT Broker 是否可用
- 防火墙是否阻止连接

### Q: 数据不显示？
**A:** 检查以下几点：
- 设备是否正在发送数据
- 数据格式是否为有效的 JSON
- Topic 名称是否匹配
- 连接状态是否正常

### Q: 如何导出数据？
**A:** 步骤如下：
1. 点击 💾 存储设置
2. 选择导出格式（CSV 或 JSON）
3. 数据将保存到应用文档目录

### Q: 蓝牙扫描不到设备？
**A:** 尝试以下方法：
- 确保手机蓝牙已开启
- 设备是否已配对
- 检查位置权限是否授予
- 尝试手动输入设备地址
